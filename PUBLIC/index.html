Sass

This was my first major project for which I developed in a Sass environment. I considered this to be, without a doubt, a game changer in how I approached development. Now I can't live without it. What I find most useful about Sass, apart from being able to designate variables, is how you're able to breakup different aspects of your site styling (layout, typography, etc.) into separate stylesheets. This is incredibly helpful when trying to keep your development environment organized, which can quickly become an issue when it comes to building a responsive site.

Reusable Stylesheets

Since this was my most extensive site yet at the time, I decided to deploy separate stylesheets for templated elements. The navbar and footer were the same across all pages, so I coded their CSS in a separate stylesheet and called them on each HTML page, as opposed to copying that code into the stylesheets for all the different page types. This was before I began using Sass in my development process, so I would approach this differently were I building the site now however, at the time I felt it was an efficient approach to consolidating my code.

HTML5 / CSS3 / JavaScript / GitHub

This was a (relatively) simple exercise implementing reusable JavaScript functions. I challenged myself to create something of game, or at least the illusion of one by writing reusable functions that apply different CSS classes. I enjoyed this exercise enormously as it provided me the chance to explore yet another means by which to consolidate and make my code more efficient.

more info

Same Function, New Classes, Different Elements

There are essentially three things on this page that change when a user clicks on the asteroid cluster. The ship fires a laser beam, the cluster breaks up, and the text in the instruction line changes. The first two changes are made by applying the same setAttribute function, and passing in arguments on the event handlers. Similarly, the text change is also handled with a reusable function.


              /* function laserBlast():

                  -Accepts ID of HTML elementID

                  -Accepts CSS class name to apply to HTML element

                  -Locates HTML by ID

                  -Changes class of target element

              */

              function laserBlast(elementID, newClass) {

                  document.getElementById(elementID).setAttribute("class", newClass);

              };

              

              /* function newInstruction():

                  -Accepts ID of HTML elementID

                  -Accepts new HTML to be applied to target element

                  -Locates HTML by ID

                  -Changes inner HTML

              */

              function newInstruction(elementID, newInstruction) {

                  document.getElementById(elementID).innerHTML = newInstruction;

              };

              

              //Click events reusing functions

              

              document.getElementById("chunk1").onclick = function() {

                  laserBlast("body", "laserBlast");

                  laserBlast("laserBeam", "fire");

                  newInstruction("instruction", "Click the Spaceship to Reload!")

              };

Plantastic

Photoshop / Adobe XD / InVision

Plantastic was an event planning application designed for UX Design at MCAD. This was built with a team of 2 other students. The process involved stakeholder interviews, competitive analysis, persona development, product requirements, information architecture, a high-fidelity prototype, as well as user testing.

more info

Wireframe & High-Fidelity Prototype

After completing user research, persona development, as well as product requirements, our team first developed a wireframe using Adobe XD, and then a high-fidelity prototype using InVision, a widely used prototyping tool throughout the web/application design industry.

Full Wireframe Full Prototype

Developing Personas and User Goals

In leading up to the development of user personas for Plantastic, we first identified patterns that were prevalent amongst real users who had event planning experience, whether in a professional context, or a personal one. The users were plotted on a variable map, and from there patterns and goals were extrapolated that later informed personas.

Full Document Full Document

Usability Testing

After deploying our high-fidelity prototype, we conducted usability testing. This provided us an excellent opportunity for feedback nearing the end of the design phase for Plantastic. We were able to successfully identify where the product fell short, which lay mostly in missing transitional states, as well as see where certain UI elements were ineffective, such as poor placement of certain buttons. In essence, I would say that more than anything, usability testing for Plantastic made clear the value of agile development, and why working in an iterative process is highly efficient.

Fancy Tiger

MailChimp / Front-End Dev / Adobe

Speculative marketing campaign for Fancy Tiger Clothing in Denver, CO. This project was completed as coursework for Digital Marketing Principles at MCAD. The development of this campaign included a digital marketing strategy, an email campaign, a blog post, as well as a direct mailer for an in-store blogger event.

more info

Email Campaign

Part one of my speculative campaign for Fancy Tiger involved developing an email campaign. I wanted to not only make announcements regarding merchandise available in their brick and mortar store, as well as their online shop, but also capture and reassert their voice and essence, which is about community, creativity, and friendship.

Email Campaign

Blog Post

I also decided to develop a blog post for the campaign, which I wanted to consist of content that highlighted Fancy Tiger's commitment to the community. By designing a shirt that brought attention to women's rights, and donating a percentage of sales thereof to an organization that fought for those rights, Fancy Tiger took on an enormously supportive role. As such, my idea was that the content of this post should highlight that role. The distribution of this post on their website, as well as through thier social media, would in turn generate awareness of not only a new product, but also an on-going fight, and an organization that is fighting it.

Blog Entry

Direct Mailer

For the final piece of my speculative campaign for Fancy Tiger, I decided to design a direct mailer that would be distributed to influential fashion bloggers in the Denver scene. The idea is that by inviting said bloggers to an in-store event, they would in turn write about Fancy Tiger online, which would then reach their followers, thus providing Fancy Tiger with greater exposure. In essence, the direct mailer would be the first step in leveraging bloggers as marketing media.
